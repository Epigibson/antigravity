## Paso 1: Descargar e instalar el CLI

Ya no es necesario clonar el repositorio ni tener Go instalado. Simplemente descarga el archivo precompilado.

1. Ve a la [página de Releases en GitHub](https://github.com/Epigibson/Nexus/releases) y descarga el archivo para tu sistema operativo.
2. Extrae el archivo (si viene comprimido) y renómbralo a `nexus` (o `nexus.exe` en Windows).
3. Configúralo en tu sistema:

<Tabs>
  <Tab title="Windows (PowerShell)">
    ```powershell
    # Instalar globalmente
    $installDir = "$env:USERPROFILE\.nexus\bin"
    New-Item -ItemType Directory -Force -Path $installDir | Out-Null
    # Mueve el nexus.exe descargado a esa carpeta
    Move-Item nexus.exe "$installDir\nexus.exe" -Force

    # Agregar al PATH (permanente, solo la primera vez)
    $currentPath = [System.Environment]::GetEnvironmentVariable("Path", "User")
    if ($currentPath -notlike "*\.nexus\bin*") {
        [System.Environment]::SetEnvironmentVariable("Path", "$currentPath;$installDir", "User")
    }
    ```

    <Warning>
      Reinicia tu terminal después de agregar al PATH para que tome efecto.
    </Warning>
  </Tab>
  <Tab title="macOS / Linux">
    ```bash
    # Dar permisos de ejecución y mover al PATH
    chmod +x nexus
    sudo mv nexus /usr/local/bin/

    # Si no tienes permisos de sudo, puedes moverlo a ~/.nexus/bin
    # mkdir -p ~/.nexus/bin
    # mv nexus ~/.nexus/bin/
    # echo 'export PATH="$HOME/.nexus/bin:$PATH"' >> ~/.zshrc
    # source ~/.zshrc
    ```
  </Tab>
</Tabs>

Verifica la instalación:

```bash
nexus version
# → Nexus v0.1.0
```

<Check>
  Si ves `Nexus v0.1.0`, la instalación fue exitosa.
</Check>

## Paso 2: Crear cuenta y API Key

<Steps>
  <Step title="Registrarte en el Dashboard">
    Ve a [nexus-production-a677.up.railway.app](https://nexus-production-a677.up.railway.app) y crea tu cuenta.
  </Step>
  <Step title="Crear un proyecto">
    En el dashboard, crea un proyecto y agrega entornos (development, staging, production). Configura los CLI profiles (GitHub, AWS, etc.) y variables de entorno (API keys, secrets).
  </Step>
  <Step title="Generar una API Key">
    Ve a **Configuración → API Keys → Generar Nueva Key**.

    <Warning>
      La key completa (`ag_live_...`) solo se muestra **una vez**. Cópiala antes de cerrar el modal.
    </Warning>
  </Step>
</Steps>

## Comandos disponibles

| Comando | Descripción |
|---------|-------------|
| `nexus login` | Autenticarte con tu API key |
| `nexus sync` | Sincronizar proyectos del cloud |
| `nexus status` | Ver estado de conexión |
| `nexus switch <proyecto> --env <env>` | Cambiar contexto completo |
| `nexus list` | Listar proyectos locales |
| `nexus init` | Crear nexus.yaml local |
| `nexus logout` | Desconectar del cloud |
| `nexus version` | Ver versión |

## Troubleshooting

<AccordionGroup>
  <Accordion title="'nexus' no se reconoce como comando">
    Asegúrate de que `~/.nexus/bin` está en tu PATH y que reiniciaste la terminal después de instalarlo.

    **Windows**: Verifica con `$env:Path -split ";" | Where-Object { $_ -like "*nexus*" }`

    **macOS/Linux**: Verifica con `echo $PATH | tr ':' '\n' | grep nexus`
  </Accordion>
  <Accordion title="Authentication failed al hacer login">
    - Verifica que copiaste la API key completa (empieza con `ag_live_`)
    - Genera una nueva key desde el dashboard si la anterior expiró
    - Verifica conexión a internet
  </Accordion>

  <Accordion title="Error 500 al generar API Key">
    Si ves un error interno al crear la key, espera 2 minutos para que Railway redeployee tras un push reciente.
  </Accordion>
</AccordionGroup>
